42 ## package depots (registries, packages, environments) ##
43
44 """
45 DEPOT_PATH
46
47 A stack of "depot" locations where the package manager, as well as Julia's code
48 loading mechanisms, look for package registries, installed packages, named
49 environments, repo clones, cached compiled package images, and configuration
50 files. By default it includes:
51
52 1. `~/.julia` where `~` is the user home as appropriate on the system;
53 2. an architecture-specific shared system directory, e.g. `/usr/local/share/julia`;
54 3. an architecture-independent shared system directory, e.g. `/usr/share/julia`.
55
56 So `DEPOT_PATH` might be:
57 ```julia
58 [joinpath(homedir(), ".julia"), "/usr/local/share/julia", "/usr/share/julia"]
59 ```
60 The first entry is the "user depot" and should be writable by and owned by the
61 current user. The user depot is where: registries are cloned, new package versions
62 are installed, named environments are created and updated, package repos are cloned,
63 newly compiled package image files are saved, log files are written, development
64 packages are checked out by default, and global configuration data is saved. Later
65 entries in the depot path are treated as read-only and are appropriate for
66 registries, packages, etc. installed and managed by system administrators.
67
68 `DEPOT_PATH` is populated based on the [`JULIA_DEPOT_PATH`](@ref JULIA_DEPOT_PATH)
69 environment variable if set.
70
71 See also:
72 [`JULIA_DEPOT_PATH`](@ref JULIA_DEPOT_PATH), and
73 [Code Loading](@ref Code-Loading).
74 """
75 const DEPOT_PATH = String[]

106 # JULIA_LOAD_PATH: split on `:` (or `;` on Windows)
107 # first empty entry is replaced with DEFAULT_LOAD_PATH, the rest are skipped
108 # entries starting with `@` are named environments:
109 # - the first three `#`s in a named environment are replaced with version numbers
110 # - `@stdlib` is a special name for the standard library and expands to its path
111
112 # if you want a current env setup, use direnv and
113 # have your .envrc do something like this:
114 #
115 # export JULIA_LOAD_PATH="$(pwd):$JULIA_LOAD_PATH"
116 #
117 # this will inherit an existing JULIA_LOAD_PATH value or if there is none, leave
118 # a trailing empty entry in JULIA_LOAD_PATH which will be replaced with defaults.
119
120 const DEFAULT_LOAD_PATH = ["@", "@v#.#", "@stdlib"]

122 """
123 LOAD_PATH
124
125 An array of paths for `using` and `import` statements to consider as project
126 environments or package directories when loading code. It is populated based on
127 the [`JULIA_LOAD_PATH`](@ref JULIA_LOAD_PATH) environment variable if set;
128 otherwise it defaults to `["@", "@v#.#", "@stdlib"]`. Entries starting with `@`
129 have special meanings:
130
131 - `@` refers to the "current active environment", the initial value of which is
132 initially determined by the [`JULIA_PROJECT`](@ref JULIA_PROJECT) environment
133 variable or the `--project` command-line option.
134
135 - `@stdlib` expands to the absolute path of the current Julia installation's
136 standard library directory.
137
138 - `@name` refers to a named environment, which are stored in depots (see
139 [`JULIA_DEPOT_PATH`](@ref JULIA_DEPOT_PATH)) under the `environments`
140 subdirectory. The user's named environments are stored in
141 `~/.julia/environments` so `@name` would refer to the environment in
142 `~/.julia/environments/name` if it exists and contains a `Project.toml` file.
143 If `name` contains `#` characters, then they are replaced with the major, minor
144 and patch components of the Julia version number. For example, if you are
145 running Julia 1.2 then `@v#.#` expands to `@v1.2` and will look for an
146 environment by that name, typically at `~/.julia/environments/v1.2`.
147
148 The fully expanded value of `LOAD_PATH` that is searched for projects and packages
149 can be seen by calling the `Base.load_path()` function.
150
151 See also:
152 [`JULIA_LOAD_PATH`](@ref JULIA_LOAD_PATH),
153 [`JULIA_PROJECT`](@ref JULIA_PROJECT),
154 [`JULIA_DEPOT_PATH`](@ref JULIA_DEPOT_PATH), and
155 [Code Loading](@ref Code-Loading).
156 """

296 const atexit_hooks = Callable[Filesystem.temp_cleanup_purge]
297
298 """
299 atexit(f)
300
301 Register a zero-argument function `f()` to be called at process exit. `atexit()` hooks are
302 called in last in first out (LIFO) order and run before object finalizers.
303
304 Exit hooks are allowed to call `exit(n)`, in which case Julia will exit with
305 exit code `n` (instead of the original exit code). If more than one exit hook
306 calls `exit(n)`, then Julia will exit with the exit code corresponding to the
307 last called exit hook that calls `exit(n)`. (Because exit hooks are called in
308 LIFO order, "last called" is equivalent to "first registered".)
309 """
310 atexit(f::Function) = (pushfirst!(atexit_hooks, f); nothing)
311
312 function _atexit()
313 while !isempty(atexit_hooks)
314 f = popfirst!(atexit_hooks)
315 try
316 f()
317 catch ex
318 showerror(stderr, ex)
319 Base.show_backtrace(stderr, catch_backtrace())
320 println(stderr)
321 end
322 end
323 end
